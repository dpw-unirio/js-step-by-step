Para salvar isso de verdade, de vez, pra sempre até o fim do mundo (ok, não é pra tanto) você precisaria de um servidor rodando um backend com um banco de dados...

Mas não priemos cânico! Existem algumas formas de salvar dados no próprio navegador do usuário.

O mais conhecido são os famigerados cookies, eles estão aí desde tempos imemoriais e são úteis para diversas coisas, porém, nos anos recentes, algumas soluções mais "amigáveis" surgiram, que é o caso do localStorage.

Tarefas:

  • Após salvar a senha, exibir o botão de salvar.
  • Criar um objeto para armazenar as informações.
  • Criar uma função que salve esse objeto no localStorage
    • O localStorage tem dois métodos principais:
      • localStorage.setItem(key, val)
      • localStorage.getItem(key)
    • O "problema" é que ele só salva strings, mas isso é facilmente contornável usando o... JSON!
    • Você pode usar o método JSON.stringify(object) para codificar um objeto qualquer para json.
  • Ao carregar a página verificar se existe um personagem salvo, se sim, preencher o formulário com suas informações.
